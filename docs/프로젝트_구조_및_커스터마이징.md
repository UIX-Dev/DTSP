## 1. Theme Color Customization

### Location

All theme colors are configured in **a single file** using CSS custom properties (`:root` variables):

```
src/DTSP_e8ight/union-twin-fe/src/assets/scss/_variable.scss
```

### Current Color Palette

```scss
:root {
  --font: #000000;              /* Primary text color (black) */
  --font-sub: #4a4a4a;          /* Secondary/subtitle text color */
  --deep-gray: #848484;         /* Deep gray for muted text */
  --gray: #aaaaaa;              /* Standard gray */
  --lightgray: #dddddd;         /* Light gray for borders/dividers */
  --lightblue: #dae2ea;         /* Light blue for subtle backgrounds */
  --bg: #ffffff;                /* Page background (white) */
  --sky: #eaeef3;               /* Sky blue for card backgrounds */
  --home-bg: #ccd4df;           /* Dashboard/home page background */
  --secondary-sky: #d7dee7;     /* Secondary sky background */
  --sidebar-hover: #dfe5ee;     /* Sidebar menu item hover state */
  --blue: #7da5d7;              /* Primary brand blue */
  --secondary-blue: #749bce;    /* Secondary brand blue */
  --yellow: #b48d52;            /* Accent yellow (used for Relationship labels) */
  --green: #558b2f;             /* Success/GeoProperty green */
  --red: #ff7043;               /* Error/warning red-orange */
}
```

### How to Change Theme Colors

Simply edit the hex color values in `_variable.scss`. The changes will apply **globally** across all 70+ SCSS files that reference these variables.

### Font Settings (same file)

The file also contains font size and weight settings:

```scss
$FONT_SIZE_REGULAR: 1.2rem;    /* Body text, inputs, selects */
$FONT_SIZE_SMALL: 1rem;        /* Labels, secondary text */

$FONT_WEIGHT_REGULAR: 500;     /* Normal text weight */
$FONT_WEIGHT_BOLD: 700;        /* Titles, navigation, emphasis */
```

### SCSS Mixins Available

| Mixin | Used For |
|-------|----------|
| `FONT_REGULAR` | Body text, input/select elements |
| `TITLE` | Page titles, navigation menu labels |
| `LABEL` | Input/select labels |
| `BOLD_LABEL` | Bold label variants |
| `TYPE_LABEL` | Entity type badges (Property, Relationship, GeoProperty) |
| `TYPE_STATUS` | Status badges (RUN, STOP, CREATED, HANG, DIED) |
| `PATH_TITLE` | Breadcrumb path titles |
| `FONT_ELLIPSIS` | Text truncation with ellipsis |
| `RESPONSIVE` | Full width/height responsive elements |
| `UNDRAGGABLE` | Prevent text selection and drag |

## 2. Logo Replacement

### Current Logo Location

The logo is displayed at the **top of the sidebar** and consists of:

1. **Icon**: `home_icon.svg` (a small house icon)
2. **Text**: `연합트윈 프레임워크` (Union Twin Framework)

**File containing the logo:**

```
src/DTSP_e8ight/union-twin-fe/src/components/sidebar/sidebar.tsx
```

### How to Replace the Logo

#### Step 1: Add Your Logo Image

Place your logo file into the images directory:

```
src/DTSP_e8ight/union-twin-fe/src/assets/images/your_logo.svg
```

> **Recommended format:** SVG (for crisp rendering at any size)  
> **Recommended size:** ~24–32px height for the sidebar

#### Step 2: Update the Sidebar Component

Open `sidebar.tsx` and make the following changes:

**a) Update the import** — Replace the HomeIcon import with your logo:

```tsx
// Before:
import { ReactComponent as HomeIcon } from '../../assets/images/home_icon.svg';

// After:
import YourLogo from '../../assets/images/your_logo.svg';
```

**b) Update the JSX** — Replace the icon and text:

```tsx
// Before:
<HomeIcon />
<span>연합트윈 프레임워크</span>

// After:
<img src={YourLogo} alt="Your Service Name" style={{ height: '28px' }} />
<span>Your Service Name</span>
```

#### Step 3: (Optional) Replace the Secondary Logo

There is also an existing logo file that may be referenced elsewhere:

```
src/DTSP_e8ight/union-twin-fe/src/assets/images/e8ight_logo.png
```

For full consistency, either:
- **Replace** `e8ight_logo.png` with your logo (same filename), or
- Search for references to `e8ight_logo` and update them to point to your new logo

#### Step 4: Update the Browser Tab Icon (Favicon)

```
src/DTSP_e8ight/union-twin-fe/public/favicon.ico
```

Replace this file with your own favicon. You can generate one from your logo at [favicon.io](https://favicon.io/).

#### Step 5: Update the Page Title

In `public/index.html`, update the `<title>` tag:

```html
<!-- Before: -->
<title>Union Twin Framework</title>

<!-- After: -->
<title>Your Service Name</title>
```

## 3. Frontend Architecture (SCSS & Component-Based)

### Architecture Pattern

The frontend follows a **Component-Based Architecture** with SCSS modules:

```
Component (.tsx)  ←→  Stylesheet (.scss)
    │                      │
    │                      ├── Imports _variable.scss (global theme)
    │                      ├── Component-specific styles
    │                      └── Uses mixins from _variable.scss
    │
    └── React component logic (TypeScript)
```

### Key Facts

| Aspect | Details |
|--------|---------|
| **Uses SCSS?** | ✅ Yes — 70+ `.scss` files |
| **Component-based?** | ✅ Yes — Each page/component has its own `.tsx` + `.scss` pair |
| **Hard-coded styles?** | ❌ No — All styles use the centralized variable system |
| **CSS Framework?** | TailwindCSS is partially used alongside SCSS |

### Example: How a Page is Structured

```
pages/
├── home/
│   ├── home.tsx          ← React component (TypeScript)
│   └── home.scss         ← Page-specific styles
├── notice/
│   ├── noticePage.tsx
│   └── noticePage.scss
└── service-description/
    ├── attribute/
    │   ├── attributeManagement.tsx
    │   └── attributeManagement.scss
    └── entity/
        ├── entityTypeModeling.tsx
        └── entityTypeModeling.scss
```

### SCSS File Organization

```
src/assets/scss/
├── _variable.scss         ← 🎨 Global theme variables (EDIT THIS FOR THEME)
├── global.scss            ← Global/reset styles
└── ...

src/components/
├── sidebar/
│   ├── sidebar.tsx        ← 🖼️ Sidebar component (EDIT THIS FOR LOGO)
│   └── sidebar.scss
├── header/
│   ├── header.tsx
│   └── header.scss
├── table/
│   ├── table.tsx
│   └── table.scss
└── ...
```

## 4. Core Features & Dashboard Overview

The DTSP portal is **not just a dashboard** — it's a comprehensive Digital Twin management platform. Here's what each major feature does:

### Dashboard (Home)

The home page displays a grid of quick-access cards linking to all platform features. This layout is customizable through the "메뉴 커스터마이징" (Menu Customizing) button in the top-right corner.

### Feature Modules

| Module | Korean Name | Description |
|--------|-------------|-------------|
| **Member Management** | 회원 관리 | User registration, role assignment (ADMIN/USER), member list |
| **Board Management** | 게시판 관리 | Notices, announcements, and bulletin board system |
| **DT Metadata Management** | 디지털트윈 메타데이터 관리 | Register, search, and visualize digital twin metadata |
| **DT Metadata Registration** | 디지털 트윈 메타데이터 등록 | Create new digital twin entity definitions |
| **Metadata Visualization Graph** | 메타데이터 시각 그래프 | Visual graph representation of metadata relationships |
| **DT Processing Management** | 디지털트윈 프로세싱 관리 | Manage digital twin data processing pipelines |
| **Service Description Tool** | 서비스 디스크립션 도구 | Entity Type Modeling, Attribute Management, Entity Query, Collection Config |
| **Prediction Tool** | 예측기 생성/연동 도구 | AI/ML model creation and integration for predictive analytics |
| **Physical Simulation Tool** | 물리 시뮬레이션 전후처리 도구 | Pre/post-processing for physics-based simulations |
| **Discrete Simulation Tool** | 이산 연속 시뮬레이션 조합 도구 | Discrete event simulation composition |
| **Union Object Sync Engine** | 연합 객체 및 동기화 엔진 관리 | Federated object synchronization across systems |
| **Data Verification Tool** | 유효성 검증 및 데이터 증강 도구 관리 | Data validation and augmentation management |
| **Dynamic Data/Event Tracking** | 동적 데이터/이벤트 트래킹 프로세스 | Real-time data and event tracking processes |
| **API & Docs Links** | API 및 문서 링크 | External API documentation and reference links |
| **3D Model Viewer** | 3D 모델 뷰어 | Interactive 3D model visualization |

## Summary

Customizing the DTSP portal to your brand requires editing only **2–3 files**:

1. **`_variable.scss`** — Change all colors in one place
2. **`sidebar.tsx`** — Replace logo icon and service name
3. **Add your logo** — Drop your SVG/PNG into `assets/images/`

No deep code changes are needed. The architecture is clean, component-based, and well-separated between logic (TSX) and styles (SCSS).
